La Lógica de las Decisiones

En la vida diaria tomamos decisiones constantemente basadas en condiciones:

  • Si está lloviendo → entonces llevo paraguas.
  • Si tengo hambre → entonces voy a comer.
  • Si la batería está descargada → entonces conecto el cargador.

En programación es igual, siempre se sigue la estructura:

Si ocurre una condición → entonces se realiza una acción.
¿Llueve?
→ Sí →
Llevar paraguas
¿Llueve?
→ No →
Ir normal
Sin estas decisiones, el software sería completamente plano y estático.

Concepto Diapositiva 5 / 23

Concepto de Operador

Un operador es un símbolo que permite realizar una operación sobre uno o más valores (variables o constantes).

¿Para qué nos sirven?
  • Sumar dos números o calcular un promedio.
  • Comparar dos edades para saber quién es mayor.
  • Comprobar si dos condiciones se cumplen al mismo tiempo (ej. usuario válido Y contraseña correcta).
Son las herramientas fundamentales que le dan dinamismo y lógica al código.

Operaciones Aritméticas en C# 

int a = 20;
int b = 5;

Console.WriteLine(a + b); // Imprime: 25
Console.WriteLine(a - b); // Imprime: 15
Console.WriteLine(a * b); // Imprime: 100

Cada operador realiza una acción distinta usando las mismas variables de origen. El comportamiento del programa cambia únicamente al cambiar el símbolo del operador.

Módulo Diapositiva 9 / 23

Comprendiendo el Operador Residuo (%)

¡Error común!

No realiza una división ordinaria. Lo que hace es devolver el residuo (sobrante) que queda después de realizar una división entera.

Ejemplos numéricos
  • 10 % 2 = 0 (10 se divide exactamente entre 2, no sobra nada).
  • 11 % 2 = 1 (cabe 5 veces y sobra 1).
  • 18 % 7 = 4 (7 × 2 = 14, y para llegar a 18 sobran 4).

Ejemplo Diapositiva 10 / 23

Aplicación: Detectar Números Pares

¿Cómo sabemos matemáticamente si un número es par? Si lo dividimos entre 2 y el residuo es exactamente 0. 

Console.Write("Ingrese un número: ");
int numero = int.Parse(Console.ReadLine());

if (numero % 2 == 0)
{
    Console.WriteLine("Es un número par");
}
Evaluamos el residuo de numero % 2. Si es igual a 0, confirmamos la paridad.

Relacionales Diapositiva 11 / 23

Operadores Relacionales (Comparación)

Sirven para comparar dos valores. Responden a preguntas lógicas: ¿Es mayor?, ¿Es menor?, ¿Es igual?, ¿Es diferente?

Regla fundamental: el resultado de cualquier comparación relacional siempre será un valor booleano: True (Verdadero) o False (Falso).

Ejemplo Diapositiva 13 / 23

Análisis de Evaluación Relacional 

int edad = 20;
Console.WriteLine(edad >= 18); // Imprime: True
¿Qué hace la computadora tras bambalinas?
  • Sustituye la variable por su valor actual: 20 >= 18.
  • Se hace la pregunta: ¿El número 20 es mayor o igual que 18?
  • Como la respuesta es afirmativa, el resultado lógico es True.

Lógicos Diapositiva 15 / 23

Operadores Lógicos

Permiten combinar varias comparaciones en una sola condición.

&& (AND / Y)

Da True únicamente si todas las condiciones individuales son verdaderas. 

edad >= 18 && tieneDNI == true
|| (OR / O)

Da True si al menos una de las condiciones es verdadera.

esViernes || esFinDeSemana
! (NOT / NO)

Invierte el valor lógico. Convierte True en False y viceversa. 

!(edad < 18)

Ejercicio Práctico 1: Cálculos y Residuos

Enunciado

Escribe un programa en C# que solicite al usuario ingresar un número entero por teclado. El programa debe verificar y mostrar en pantalla si el número ingresado es múltiplo de 5 o no lo es.

Teoría aplicada: un número es múltiplo de 5 si al dividirlo entre 5, la división es exacta (su residuo es 0). 
Console.Write("Ingrese un número entero: ");
int numero = int.Parse(Console.ReadLine());

if (numero % 5 == 0)
{
    Console.WriteLine("El número es múltiplo de 5.");
}
else
{
    Console.WriteLine("El número NO es múltiplo de 5.");
}

if / if-else Diapositiva 17 / 23

Control de Flujo: if e if-else

if simple

Ejecuta un bloque de instrucciones si la condición es verdadera. Si es falsa, simplemente lo ignora.

if-else doble

Define un camino a seguir cuando la condición se cumple (if) y un camino alternativo obligatorio para cuando no se cumple (else). 

if (condicion) {
    // Código si es verdadero
} else {
    // Código si es falso
}

Ejercicio Práctico 2: Selección Múltiple (if-else if)

Enunciado

Desarrolla un sistema que pida a un estudiante su calificación final de un examen (rango 0 a 20) e imprima el desempeño:

Console.Write("Calificación (0-20): ");

    Console.WriteLine("Excelente");
} else if (nota >= 14) {
    Console.WriteLine("Bueno");

    Console.WriteLine("Regular");
} else {
    Console.WriteLine("Deficiente");
}

switch Diapositiva 19 / 23

Estructura de Control: switch

Se utiliza para simplificar el código cuando tenemos que comparar una única variable contra múltiples valores enteros, caracteres o cadenas constantes. Evita tener que escribir demasiados else if seguidos.

case [valor]

El caso que queremos comparar.

break;

Obligatorio para salir del bloque del switch tras ejecutar un caso.

default

Se ejecuta si ningún caso anterior coincide (equivalente al else).

Ejercicio 3 Diapositiva 20 / 23

Ejercicio Práctico 3: Aplicando switch

Enunciado

Solicita dos números reales y muestra un menú (1. Sumar, 2. Restar). El usuario elige una opción y el programa ejecuta el cálculo correspondiente usando switch. Cualquier otro valor muestra "Opción inválida". 

Console.Write("Num 1: "); double n1 = double.Parse(Console.ReadLine());
Console.Write("Num 2: "); double n2 = double.Parse(Console.ReadLine());
Console.Write("Elija opción (1-Suma, 2-Resta): "); int op = int.Parse(Console.ReadLine());

switch (op) {
    case 1:
        Console.WriteLine($"Suma: {n1 + n2}");
        break;
    case 2:
        Console.WriteLine($"Resta: {n1 - n2}");
        break;
    default:
        Console.WriteLine("Opción inválida.");
        break;
}

Console.Write("Promedio: "); double promedio = double.Parse(Console.ReadLine());
